## 2. Текущий нарративный поток

S1 **Hook** (это живая мем-арена, хочу внутрь) → S2 **шторы раскрываются** взрывом, начинается погружение «как играть» → S3 **проживаешь матч** (глубокий сторителлинг) → S4–S5 **доказательство жизни** (живые матчи + тикер) → S6 **что это** коротко → S7 **бойцы** (хочу этим играть) → S8–S9 **конкуренция + трейлер** (слава, кульминация чувств) → S10 **roadmap** (живой продукт) → S11–S13 **доверие + экономика + кланы** (матрица-слой: честность, токеномика, реф-граф) → S14 **FAQ** (снять возражения) → S15 **финальный CTA** (войди) → S16 footer.

Дуга: **присутствие → погружение → доказательство → желание → слава → доверие → решение.**

## 3. Целевая структура (куда идём сейчас)

- **Остаётся** порядок и роли секций (S1→S16) — он соответствует SPEC и progressive disclosure.
- **Эволюционирует в кино-многослойность** (как Hero сейчас): каждый блок — не «контент+картинка», а сцена с 3 z-планами (фон-атмосфера → средний план → передняя вуаль), пересечением слоёв, контролируемым светом, frame-break.
- **Вайб-дисциплина:** каждая секция держит ОДИН ведущий вайб (см. колонку), но на единой тёмной пиксель-мем базе → скролл = путешествие по настроениям.
- **Большие story-биты (приоритет докрутки):** S1 Hero ✅ → **S2/S3 колизей-погружение** (следующий фокус) → S7 ростер (казино) → S4 live-arena (стена мониторов) → S12 экономика-печь → S13 реф-граф.
- **Progressive disclosure усиливается:** ранние блоки = эмоция/присутствие (минимум текста), детали (provably-fair, токеномика, реф-формулы) — глубже и раскрываются по скроллу/клику, не вываливаются сразу.

## 4. Гид по скриншотам (как называть блоки в фидбэке)

Когда присылаешь скрин — называй так, я сразу пойму место в потоке:
- **«Hero»** = S1 (заголовок FIGHT/EXPLODE/GET PAID + broadcast-монитор).
- **«Curtain / Как играть интро»** = S2 (чёрные шторы → reveal на фоне колизея).
- **«Story Chapters / геймплей-сторителлинг»** = S3 (глубокое погружение в матч).
- **«Live Arena / стена мониторов»** = S4. **«Тикер»** = S5. **«Bento / что это»** = S6.
- **«Ростер / бойцы»** = S7. **«Лидерборд»** = S8. **«Трейлер»** = S9. **«Roadmap»** = S10.
- **«Provably Fair»** = S11. **«Экономика / печь»** = S12. **«Кланы / граф»** = S13.
- **«FAQ»** = S14. **«Финальный CTA»** = S15. **«Footer»** = S16.

## 5. Пробелы и неоднозначности

- **S4 Live Arena Stage — тонкая (47 строк)** vs SPEC §4 (control-room, сетка мониторов 4×2, 3D-perspective). Сейчас не дотягивает до «доказательства жизни». Кандидат на докрутку.
- **S13 Guilds — только тизер (70 строк)** vs SPEC §12 (D3 force-граф + live-калькулятор). Полноценный реф-граф ещё не реализован.
- **S12 Economy — печь недо-собрана (85 строк)** vs SPEC §11 (animated forge, потоки монет, burn). Образ «печи/token sink» пока слабый.
- **S6 Bento Scene — вне SPEC** (доп. «что это»-обзор). Надо решить: оставляем как мост S5→S7 или растворяем в соседях (риск дублирования с S3/S7).
- **S7 Roster (128 строк)** — есть, но SPEC §6 хочет horizontal-scroll pinned + ауры + Rive. Сейчас, вероятно, статичная сетка. Под вайб «казино/памп» — докрутка.
- **Двойная палитра:** SPEC задаёт OKLCH (`--color-void` и т.д.), код живёт на hex (`#f5c842`/`#3a9e9e`/`#d44030`). Держать lockstep, не плодить два набора (зафиксировано в SPEC прим. к §0).
- **Имя S2:** SPEC зовёт §2 «Curtain Reveal», но внутри сейчас «How to play» интро (`ArenaStoryReveal`) — фактически это шов §2-механика + начало §3-нарратива. В фидбэке зовём **«S2 / Curtain → Как играть»**.
- **Другой агент** держит на `origin/main` иную версию лендинга — наша сессия локальная (не коммитим). Расхождение надо будет однажды свести.
