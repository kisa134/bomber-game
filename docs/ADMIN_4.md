# Admin 4.0 — спецификация (источник правды)

Цель: **единый, data-driven центр управления проектом** на стеке Кирилла
(React/shadcn/recharts), на русском, **только реальные данные**, с ИИ-ассистентом
как полноценным чатом и **живой интерактивной картой архитектуры/потоков данных**.
Не «вайбкодинг» — серьёзная сквозная аналитика и система управления бэком.

Принципы:
1. **Новая админка = главная.** `/admin` отдаёт новую (на стеке Кирилла). Старую
   ванильную убираем; всё полезное из неё уже перенесено в ops-страницы. Никаких
   «нажми Marketing Hub в углу».
2. **Только реальные данные.** Любой экран либо показывает живые данные из нашего
   бэкенда/подключённого API, либо честно помечает «источник не подключён» (а не
   фейк). Демо-данные Кирилла (соц-подписчики, клипы, KOL, новости) — это **моки**,
   их либо подключаем к реальным API, либо прячем за бейджем «Demo / not connected».
3. **Русский интерфейс** (где уместно; тех-термины можно латиницей).
4. **Каждый экран самообъясняем** — короткий «что это / что делать», ИИ может
   пояснить и подтянуть нужное.

## A. Подключения и источники данных
**Страница «Подключения» (Connections / Health):** статус каждого шлюза вживую:
- 🟢/🔴 Postgres, Solana RPC, Treasury-кошельки, Telegram bot, Google OAuth,
  PostHog, GA4, Clarity, ИИ-провайдер, DexScreener (цена токена).
- Для каждого: подключён? последний успешный ответ? задержка? поток данных идёт?
- Реализация: сервер `/admin/connections` пингует каждый источник и отдаёт статус.

**Соц-API (для реальных маркетинг-цифр) — нужно подключить (твоя зона — доступы):**
- X/Twitter API (нужен developer-доступ — запросить), TikTok, Instagram Graph,
  YouTube Data API, Telegram. Пока не подключены → маркетинг-страницы помечаем
  «Demo». Когда дашь ключи — заменю моки на реальные фетчи.

## B. ИИ-ассистент (полноценный чат)
- **Чат как в топовых ИИ-приложениях:** диалог с историей, стриминг ответа,
  видно «думает / тянет данные / какой инструмент вызвал» (tool-calls).
- **Знает весь проект:** контекст = `/admin/snapshot` (экономика, игроки, матчи,
  турниры, нагрузка, ошибки, рефералка) + **файловая память** (логи ниже).
- **Инструменты (tools):** «открой экран X», «сведи аналитику Y», «покажи топ/казну/
  турниры», «дай метрику Z за период».
- **Проактивно 2×/день** (cron): краткий бриф «на что обратить внимание сегодня —
  тех / маркетинг / аналитика; какие задачи поставить». Пишется в лог + показывается
  на дашборде.
- **Память/логи (файлы):** ежедневный Markdown-лог состояния (`docs/ai-log/YYYY-MM-DD.md`
  или в БД) — ИИ читает прошлые логи, ведёт непрерывную картину. Точка для твоего
  внешнего ИИ с памятью.
- **Нужно от тебя:** ключ ИИ на Render (`BOMBER_ADMIN` или `AI_API_KEY`+`AI_PROVIDER`).

## C. Живая карта архитектуры / потоков данных (must-have)
- Интерактивная блок-схема: все сущности (клиент, сервер, комнаты, матчмейкер,
  экономика, казна, рефералка, БД, Solana, Telegram, аналитика, ИИ) и **связи/
  потоки** между ними.
- **Живая:** узлы подсвечиваются по статусу (🟢 ок / 🟡 нагрузка / 🔴 ошибка),
  на рёбрах — поток (запросы/сек, объём). Клик по блоку → его метрики/логи/экран.
- Источник — реальный код + `/admin/connections` + `/admin/stats`. Не картинка —
  живой дашборд «как дышит проект».

## D. Data-driven логирование (фундамент)
- Чётко определить **сущности и атрибуты** (player, match, room, tournament,
  deposit/withdraw, referral, spin, session) и **метрики** (DAU, retention, ARPU,
  payin/payout, rake, виральность и т.д.).
- Всё корректно пишется в БД (event-лог) → регулярный анализ в цикле (ИИ + дашборды).
- Логи турниров, экономики, ошибок — для последующего улучшения.

## E. Экраны (русский + сценарии)
Sidebar (RU): **Пульт · Игроки · Деньги · Турниры · Система · ИИ-ассистент ·
Карта проекта · Подключения · Маркетинг(Demo до API)**. У каждого — строка
«Зачем этот экран / что делать», ИИ-кнопка «объясни/помоги».

## Фазы разработки Админки 4.0
1. **Новая = главная + RU + честные данные:** `/admin` → новый хаб; перевод;
   моки-страницы под бейдж «Demo»; страница «Подключения» (статусы шлюзов).
2. **ИИ-чат:** стриминг + tool-calls + история + файловый лог (память) + проактивные
   2×/день брифы (cron). (Нужен ключ ИИ.)
3. **Карта проекта:** живая интерактивная блок-схема потоков данных.
4. **Data-driven:** доопределить сущности/метрики, event-лог, цикл анализа.
5. **Соц-API:** реальные маркетинг-цифры (по мере доступов).

## Что нужно от тебя
1. Ключ ИИ на Render (для фаз 2-3).
2. Доступы к соц-API (X dev и т.д.) — для реальных маркетинг-цифр (фаза 5).
3. Подтвердить фазы/приоритет.

## Турниры 4.0 (отдельный пласт, после админки)
Автоматизированные, как в нормальных сетевых играх:
- **Авто-расписание:** новый турнир каждые N минут; при наборе нужного числа
  регистраций — **запускается автоматически** (без ручных кнопок).
- **Хаб → «Турнир» → отдельный полноэкранный режим:** список участников и как он
  заполняется, **турнирная сетка** с продвижением и результатами матчей, кнопка
  «в матч», красивая живая визуализация.
- **ИИ следит** за турнирами, всё логируется (для улучшения).
