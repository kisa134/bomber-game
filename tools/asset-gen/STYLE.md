# asset-gen — STYLE & COMPOSITION bible

Цель: генерить **в нашем стиле**, а не «AI-slop». Два рычага: (1) жёсткие стиль-якоря в `gen.ts`,
(2) **референс-изображения наших ассетов** через `--ref` (edit-модель `nano-banana-2/edit` копирует
наш look). Без референсов модель сочиняет генерик-3D — поэтому `--ref` почти всегда.

---

## 1. Визуальная ДНК (что всегда / чего никогда)
**Всегда:** premium 2D pixel-art-влияние, hand-crafted arcade, мем-бойцы с большими головами,
графит-камень-металл арена, cyan tech-швы, золото/ember-акценты, **глубокий чёрный**, плоский
кинематографичный свет, чистый читаемый pixel/vector силуэт, дорого и сдержанно.
**Никогда:** generic 3D / octane / unreal render, фотореализм, аниме, painterly масло, мыло/blur,
soft AI-slop, текст/лого/водяные знаки/UI поверх.

**Палитра:** gold `#f5c842` · teal `#3a9e9e` · red `#d44030` на void-чёрном. Без синего/фиолетового.

## 2. Референсы (что подавать в `--ref` под каждый тип)
Пути от `tools/asset-gen/`:
| Хочу сгенерить | Подаю как референс (`--ref`) |
|---|---|
| Бойца / силуэт / экшн-кадр | наш спрайт героя: `../../apps/landing/public/sprites/skin_2.webp` (trump), `skin_3.webp` (elon), `pepe/…`, `doge/…` |
| Арена-плита / окружение / блоки | спрайты мира игры: блоки/explosion `../../apps/landing/public/sprites/explosion_0.webp`, powerup_*, + готовый `public/bg/hero-arena-a.webp` |
| FX (взрыв/дым/искры) | `explosion_0..4`, `powerup_fire.png` |
| Композ-сцена | 1 герой + 1 арена + 1 fx (до 2–3 референсов разом — edit-модель миксует стиль) |

> Правило: **минимум 1 наш референс** на каждую генерацию для людей/арены/fx. Чистый T2I — только
> для абстрактных слоёв (дым, градиент-туман, grain), где стиль персонажей не нужен.

## 3. Система МНОГОСЛОЙНОЙ композиции (как «мультик», глубина в коде)
Каждую сцену лендинга собираем НЕ одной картинкой, а **стопкой прозрачных слоёв** → параллакс/маски
в коде = реальная глубина и анимация. Слои (от дальнего к ближнему):

```
L0  BACKDROP      — атмосфера: void + сетка + дальняя арена/туман   (16:9, непрозрачн., parallax медленно)
L1  ENVIRONMENT   — сцена/пол арены/блоки/трибуны                   (16:9, можно с альфой по краям)
L2  SUBJECT       — бойцы / бомбы / момент (экшн-кадр)              (PNG альфа, --cutout)
L3  FX            — взрыв / дым / искры / debris                    (PNG альфа, --cutout, аддитивно)
L4  FOREGROUND    — ближний боец/обломок/HUD-плита (breakout)       (PNG альфа, --cutout, parallax быстро)
```
**Принцип:** каждый слой — отдельный ассет в нужном **ракурсе**; в коде они перекрываются, едут с разной
скоростью, маскируются друг другом. Это даёт depth+движение без видео. (Видео-плиты — опционально,
где нужен живой loop.)

### Ракурсы (angles) под наши сцены
- **Hero монитор / Live Arena:** spectator 3/4 сверху (как стадион-трансляция).
- **Storytelling §3 «поле»:** near-top-down (как геймплей-сетка).
- **Шторки/CTA:** боковой профиль бойцов (бег навстречу) — у нас уже есть `*_side_*` спрайты как референс.
- **Sudden death / взрыв:** низкий драматичный угол, центр-взрыв.

## 4. Каталог ассетов под секции (что генерим, слой, ракурс)
| Секция | Нужные слои/ассеты | Ракурс |
|---|---|---|
| Hero монитор | L0 арена-backdrop · L2 боец-инсет · L3 искры | 3/4 spectator |
| §2 шторки | L4 раннеры (есть side-спрайты) · L3 взрыв-вспышка · дым | side profile |
| §3 Story (5 бит) | L0 env-плита (общая, sticky) · L2 момент на бит · L4 fg-силуэт | top-down / 3/4 |
| Live Arena | L0 control-room · L1 стена мониторов · L3 ambient дым | 3/4 perspective |
| Leaderboard | L1 джумботрон-рамка · L3 свет топ-3 | front |
| Economy | L2 печь/forge · L3 пламя/искры (есть fire-спрайт референс) | front |
| CTA/Footer | L4 персонажи на кромке (side/idle) · L0 трибуны | side |

## 5. Воркфлоу
1. Берём слот из таблицы → пишем **короткий subject-промпт** (стиль-якоря добавятся сами).
2. Подаём **наши референсы** (`--ref …`) под тип.
3. `--cutout` для L2/L3/L4 (прозрачный PNG для слоёв).
4. Выход `out/*.webp` → ревью → копируем в `apps/landing/public/bg|sprites|fx/` → подменяем слот.
5. Если слабо — добавить ещё референс / уточнить ракурс, **не менять стиль-якоря**.

> Держать в lockstep со `apps/landing/SPEC.md`. Стиль-якоря менять только осознанно.

## 6. МНОГОСЛОЙНЫЕ АНИМИРОВАННЫЕ СЦЕНЫ (gen → сборка)
Сцена = стопка прозрачных слоёв, собранная движком **`apps/landing/components/scene/LayeredScene.tsx`**
(параллакс по скроллу + idle-движение + FX-блендинг). Цикл: **спланировал слои → нагенерил по пресетам
→ вырезал → описал спекой → движок собрал и оживил.**

### Шаг 1 — декомпозиция сцены (что вообще нужно)
По любой сцене спрашиваю себя 5 вопросов → 5 слоёв:
| Слой | Вопрос | Пресет gen | cutout | motion | blend | depth |
|------|--------|-----------|--------|--------|-------|-------|
| L0 backdrop | какая атмосфера/мир позади? | `env`/`control` или CSS | нет | `drift` | — | −0.6 (медленно) |
| L1 environment | какая арена/пол/сцена? | `env`/`field` | по краям | `none` | — | −0.2 |
| L2 subject | кто/что главное (бойцы, момент)? | `fighter` | **да** | `float` | — | 0 |
| L3 fx | какие спецэффекты накидываются? | `fx` (взрыв/дым/искры) | **да** | `pulse` | `screen`/`plus-lighter` | +0.3 |
| L4 foreground | что ближе всех, выступает за край? | `fighter`/`fx` | **да** | `drift` | — | +0.6 (быстро) |

### Шаг 2 — генерация слоёв
`npx tsx gen.ts --preset <тип> --name <scene>-<layer> --draft --prompt "..."` (+ `--cutout` для L2–L4).
Draft-first → отбор → финал по `--seed`. Ракурс держать единым на всю сцену (см. §3 «Ракурсы»).

### Шаг 3 — сборка (спека для `<LayeredScene>`)
```tsx
<LayeredScene className="h-[100svh]" layers={[
  { src: "/bg/arena-env.webp",   z: 0,  depth: -0.6, motion: "drift" },          // L0
  { src: "/bg/arena-floor.webp", z: 10, depth: -0.2 },                            // L1
  { src: "/fx/fighter-a.webp",   z: 20, depth: 0,    motion: "float", pixel:true },// L2 (cutout)
  { src: "/fx/explosion.webp",   z: 30, depth: 0.3,  motion: "pulse", blend:"screen" }, // L3 (cutout, additive)
  { src: "/fx/fg-fighter.webp",  z: 40, depth: 0.6,  motion: "drift", pixel:true,
    style:{ left:"-6%", bottom:"-8%", width:"38%", height:"auto", inset:"auto" } }, // L4 breakout
]} />
```
Движок сам: расставит z, даст параллакс по `depth`, оживит по `motion`, смешает FX по `blend`,
и на `<768px`/reduced-motion соберёт статичный стек (атмосфера сохранена).

### Правила
- **FX-слои** (взрыв/дым/искры) — всегда `--cutout` + `blend:"screen"`/`"plus-lighter"` (аддитивно на чёрном).
- **Глубина** разводит планы: bg `depth<0` (медленно/назад), fg `depth>0` (быстро/вперёд).
- **Единый ракурс** всех слоёв сцены, иначе развалится перспектива.
- Анимации: **по структуре, не ради ряби** (drift медленный, pulse мягкий).
- Скриншоты от владельца (другие арены/трава/блоки) → кидать в `--ref` для ещё точнее стиля.
